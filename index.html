JDT/Core feature highlights [1]

  • Java Model, AST
  • Parser
  • Error-tolerant, incremental compiler (ecj)
  • No UI dependency
  • No version dependency on any JDK (1.4 and up)
[1] JDT Core - Eclipse Wiki

Running ecj from the command line [2][3]

$ java -jar org.eclipse.jdt.core_<version>.jar
  • Available separately as ecj.jar
  • o.e.j.internal.compiler.batch.Main:3851
  • Programmatic access
  • Support for ant
[2] The Coder Lounge: ECJ - Eclipse Compiler for Java
[3] Eclipse documentation - Using the batch compiler

Extension = Code modification

Grammar changes [4]

  • Uses the Jikes parser generator [5]
  • Modify java.g
  • Regenerate and update pieces of the codebase
  • Hope for the best
[4] JDT Core / HowTo: Generate the Parser
[5] IBM Jikes Compiler for the Java Language

Grammar changes (cont.)

Changing the parser is a risky operation if you miss one of the steps above. The resulting parser can be completely unpredictable.

Extension = Instrumentation

Project Lombok [6][7]

  • Annotation processing (APT)
  • Can be used with plain javac, NetBeans, Eclipse
  • Minimal runtime component
  • delombok: reversible
  • IDE support: java instrumentation agent[8]
